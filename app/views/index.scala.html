Multi-tenant X-Authentication Service
Multi-tenancy
Allows to handle multiple customers.
Workspaces
Workspaces are the main user containers, they belong to one tenant.
System Tenant
The first workspace container.
System Workspace
The first workspace container that makes the application ready to work.
Data Isolation
Each workspace stores all user data into dedicated database.
Trusted Clients
Multiple clients can be configured into a workspace and they allow to access to the REST APIs.
JWT
XAuth allows to handle the sign-in through the JSON Web Token (JWT) open standard (RFC 7519).
JWK
Support for JSON Web Key (JWK) open standard (RFC 7517).
Multiple Encryption Algorithms
XAuth supports HMAC, Asymmetric, RSA and ECDSA encryption algorithms: HMD5, HS224, HS256, HS384, HS512, RS256, RS384, RS512, ES256, ES384 and ES512.
Configurable token durations
Access and Refresh token durations can be configured at workspace level.
Applications and Permissions
Applications can be configured at workspace level to custom user accesses for your ecosystem.
Applications
Applications can be configured at workspace level to custom user accesses.
SYSTEM Role
Allows to recognise all system administrators that are enabled to create Tenants, Workspace and manage them.
ADMIN Role
Allows to recognise all workspace administrators, they can handle other user's grants.
User lifecycle
The system handle the user flows for registration, password reset and so on.
REST APIs
The system expose REST APIs with JSON format.
Validated REST API's
The system validates all request with JSON Schema, this allows to Front-end engineers to consult public exposed schema.
